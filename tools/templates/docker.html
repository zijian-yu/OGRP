Docker

Purpose of using docker technology

        Using docker technology we can create the analysis process of the tools we build to be used locally. And this is system independent and can be run on many platforms. Whether it's a physical machine, a virtual machine, a public cloud, a private cloud, or a laptop, the results are all consistent.

How to directly deploy the generated Docker image

Use the Docker save/load command: 

# save image to tar locally
docker save -o ./image.tar $IMAGEID
# copy to target host
scp ./image.tar user@host:~/
# load into target docker repo
ssh user@host "docker load -i ~/image.tar"
# tag the loaded target image
ssh user@host "docker tag $LOADED_IMAGE_ID myimage:latest"

PS: LOADED_IMAGE_ID can be retrieved in following way: 

REMOTE_IMAGE_ID=`ssh user@host"docker load -i ~/image.tar" | grep -o "sha256:.*"`

Update: You can gzip output to make it smaller. 

docker save $IMAGEID | gzip > image.tar.gz
